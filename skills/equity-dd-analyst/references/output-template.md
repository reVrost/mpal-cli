# Equity DD Output Template

Use this structure unless the user asks for a different format.

## Simple Conclusion

- **Best risk/reward:** `<ticker>` - one sentence.
- **Best quality:** `<ticker>` - one sentence.
- **Most speculative:** `<ticker>` - one sentence.
- **Wait/avoid:** `<ticker>` - one sentence.
- **Confidence:** High / medium / low, with one reason.

## Scorecard

Scores are 1-5 where 5 is best, except execution risk where 5 means low risk.
Use the default thematic weighting unless the user specifies otherwise: 30% exposure purity, 25% valuation attractiveness, 20% financial resilience, 15% growth visibility, and 10% execution risk. Downside protection and evidence confidence are override checks that can cap the final verdict.
Recalculate the weighted overall values before finalizing; if a judgment override changes the ranking, state the override plainly.

## Company Notes

For each ticker:

- **Business and exposure:** factual description with source.
- **Financial position:** revenue, margin, cash flow, leverage, liquidity, and valuation read.
- **Valuation:** EV/EBITDA or EBITDAF, P/E, FCF yield, market cap/EV, and peer/context comparison where available.
- **Valuation stress:** one compact sensitivity or downside bridge for the main valuation driver.
- **Data-center relevance:** direct, indirect, or weak; explain why.
- **Ownership/funding mechanics:** for holding companies, show look-through stake/value and funding sensitivity; for contractors, show backlog-to-revenue, working capital/cash conversion, bank guarantee/bonding capacity, contract type, and margin volatility where disclosed.
- **Evidence confidence:** high, medium, or low, and why.
- **Bull case:** 2-3 bullets.
- **Bear case:** 2-3 bullets.
- **Key evidence:** cite primary sources close to the facts they support, not only in the source notes.

## Cross-Company Read

Compare the names on exposure purity, growth quality, funding risk, valuation, and catalysts. State which differences matter most.

## What Would Change The View

List the top three concrete checks, filings, contract wins, funding events, or price/valuation changes that would change the conclusion.

## Source Notes

List the most important sources with dates. Flag stale, missing, or secondary-only data.
Separate company-filed facts from Marketpal-derived facts. If a secondary source was used because a primary document was unavailable, name the primary document to verify next.
